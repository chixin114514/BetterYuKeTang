# Current Information Gaps

## 1. 页面范围与入口仍不明确

当前仍不清楚：

- 登录后进入课程的标准路径
- 插件要覆盖的页面清单
- 课程列表页、课程详情页、内容列表页是否为独立页面
- 页面切换是否通过路由切换还是完整跳转

这会影响：

- 页面识别设计
- 脚本注入范围
- 模块初始化时机
- SPA 监听策略

## 2. 课程内容列表结构未知

当前仍不清楚：

- 作业、实验、PPT 是否在同一个列表中
- 内容项是否有稳定类型标识
- 列表是否分页、分章、折叠、懒加载
- 状态、deadline、完成度是否能在列表中直接拿到

这会影响：

- 分类能力
- 聚合能力
- 快速筛选和跳转
- deadline 汇总

## 3. 作业 / 实验状态来源未知

当前仍不清楚：

- 状态是在列表页可见，还是必须点进去才能拿到
- 是否有截止时间、提交状态、评分状态、逾期状态
- 作业与实验是否跳转到不同子系统
- 状态字段是否统一

这会影响：

- 第一优先级需求能否在一期完整落地
- 需要做“列表聚合”还是“详情补采集”
- 是否需要跨页面抓取信息

## 4. PPT 阅读页结构与完成判定未知

当前仍不清楚：

- 当前页和总页数在哪里显示
- 进度 100% 如何判定
- 是否有未读页标识
- 页面是翻页式还是滚动式
- 页面内容是图片、canvas、PDF 还是 HTML
- 是否存在目录、缩略图、页码输入框

这会影响：

- PPT 可视化
- 自动跳到未读页
- 自动化是否可行
- 风险等级判断

## 5. 打印链路未知

当前仍不清楚：

- 打印按钮在哪个页面
- 点击打印后是否打开新页面
- 打印内容是否完整
- 是否要先加载全部页
- 浏览器打印时当前实际痛点是什么

这会影响：

- 第二优先级需求的实现方式
- 是做入口增强，还是做预检查，还是做打印视图辅助

## 6. 批量处理能力的实际边界还不够清楚

虽然已经确认“希望支持”，但当前仍不清楚：

- 批量打开最多希望一次开多少个
- 是否接受分批打开
- 批量导出课件是“目标设想”还是“明确一期目标”
- 是否需要批量处理前的确认面板

这会影响：

- 批量功能的交互与风险控制
- 浏览器弹窗限制规避方案
- 默认策略设计
