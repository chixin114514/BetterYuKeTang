# Phase 1 Overview

## 一期目标定位

一期项目边界已经明确为：

- 页面增强
- 流程辅助
- 部分自动化

同时满足以下约束：

- 不以“自动完成学习任务”为默认目标
- 可以接受少量自动化，但必须可控
- 自动化能力必须有日志
- 用户必须能明确知道插件做了什么、何时做的、做到哪一步

## 一期设计原则

- 优先解决高频核心痛点
- 保留后续可扩展结构
- 兼顾效率、稳定性和风险控制
- 将功能按自动化敏感程度和落地依赖进行分层

## 一期功能分层标准

### 必须做

满足以下任一条件的功能优先归入该层：

- 直接对应当前最高优先级痛点
- 是其他功能的基础设施
- 对一期可用性和可维护性有决定性影响

### 应该做

满足以下特征的功能归入该层：

- 明确有价值
- 与已确认痛点高度相关
- 依赖更多页面信息或实现复杂度略高于基础功能

### 可选做

满足以下特征的功能归入该层：

- 有明显增益，但不影响一期核心闭环
- 更偏增强体验或个性化

### 暂缓做

满足以下特征的功能归入该层：

- 自动化敏感度高
- 账号风险、规则风险或稳定性风险偏高
- 当前页面与业务信息不足，暂不适合在一期承诺
