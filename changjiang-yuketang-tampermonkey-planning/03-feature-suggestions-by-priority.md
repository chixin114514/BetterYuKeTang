# 建议增加的功能列表（按优先级排序）

以下功能建议均以“贴合学生使用场景”“油猴脚本有现实可行性”为筛选原则，并明确说明价值、适用场景、实现难点、前置条件及需先确认的信息。

## P1：课程内任务统一筛选与状态看板

**类别**：核心功能，纯页面增强

**价值**：

- 让用户快速看清“哪些是 PPT、哪些是作业、哪些是实验、哪些还没完成”
- 直接减少查找成本

**适用场景**：

- 一个课程内内容很多、列表杂乱
- 学生只想优先处理“待完成项”

**实现难点**：

- 内容类型是否能稳定从标题、标签、图标、DOM 结构中识别

**前置条件**：

- 需要提供课程内容列表页截图或 HTML 片段

**需先确认**：

- 列表是否分页
- 是否存在不同教师使用不同模板的情况

## P2：PPT 阅读页进度识别与阅读状态可视化

**类别**：核心功能，流程辅助

**价值**：

- 让用户知道当前页、总页数、已读状态、距离 100% 还差哪些页
- 比平台默认体验更可控

**适用场景**：

- 需要完成 PPT 学习进度
- 不确定是否已真正记录完成

**实现难点**：

- 平台如何判定“已读”
- 是否按页触发、按停留时间触发、按滚动触发、按接口上报触发

**前置条件**：

- 需要 PPT 阅读页截图
- 需要阅读页 DOM 片段
- 最好提供打开 DevTools 后与进度相关的网络请求信息

**需先确认**：

- 阅读页是翻页式、滚动式，还是缩略图+主视图结构

## P3：打印入口识别与导出前检查

**类别**：核心功能，流程辅助

**价值**：

- 帮助用户少找入口、少踩坑
- 在导出前提示“是否已经加载全部页”“是否建议切换打印参数”

**适用场景**：

- 经常需要把 PPT 保存为 PDF
- 平台打印能力可用但流程繁琐

**实现难点**：

- 打印功能是否在 iframe / 新窗口 / 独立预览页中
- 打印内容是否会因懒加载导致缺页

**前置条件**：

- 需要提供打印入口所在页面截图
- 需要说明点击后发生什么
- 最好给出打印预览页 HTML 结构或截图

## P4：未完成内容快速导航

**类别**：增强功能，纯页面增强 / 流程辅助

**价值**：

- 一键跳到未完成 PPT、未提交作业、未完成实验
- 减少课程内来回查找

**适用场景**：

- 单课程内容较多
- 多种任务混在一起

**实现难点**：

- 需要准确识别“未完成”的视觉标记或状态字段

**前置条件**：

- 需要内容列表页结构与状态样式信息

## P5：课件阅读中的快捷操作面板

**类别**：增强功能，纯页面增强 / 流程辅助

**价值**：

- 为阅读页补齐更顺手的操作入口
- 适合油猴脚本落地

**可包含但需后续确认的能力**：

- 显示当前页/总页数
- 上一页/下一页快捷入口
- 跳转到指定页
- 标记已访问页面
- 只显示辅助状态，不干预平台逻辑

**实现难点**：

- 页面是否有现成翻页 API 或可稳定触发的按钮
- SPA 页面刷新时是否会导致面板丢失

**前置条件**：

- 需要阅读页结构

## P6：课件导出辅助提示面板

**类别**：增强功能，流程辅助

**价值**：

- 在打印前提示导出建议
- 降低导出 PDF 后页缺失、样式错乱、分页异常的概率

**适用场景**：

- 平台打印功能存在可用但不稳定的情况

**实现难点**：

- 浏览器打印行为受浏览器本身限制
- 油猴脚本只能辅助，不能完全控制用户系统层打印流程

**前置条件**：

- 需要实际打印流程截图

## P7：课程级学习概览

**类别**：增强功能，纯页面增强

**价值**：

- 快速看一个课程内：
  - 总任务数
  - 未完成任务数
  - PPT 数量
  - 作业/实验数量
- 更适合周度规划

**实现难点**：

- 课程内容是否完整加载
- 是否需要滚动加载全部内容才能统计完整

**前置条件**：

- 需要课程详情页样式与加载方式

## P8：批量打开待处理内容

**类别**：可选功能，流程辅助，需谨慎

**价值**：

- 对“集中处理多个 PPT / 作业入口”有帮助

**适用场景**：

- 想快速建立处理队列

**实现难点**：

- 浏览器弹窗限制
- 平台是否限制多标签页并发
- 容易引发体验问题

**前置条件**：

- 需要确认是否真的需要批量处理
- 需要知道平台多标签行为是否稳定

## P9：本地学习记录补充层

**类别**：可选功能，纯页面增强

**价值**：

- 在平台之外，本地记录看过哪些内容、哪些内容主观认为已处理
- 避免平台状态不透明

**适用场景**：

- 平台状态显示不可靠
- 希望有自己的标记体系

**实现难点**：

- 本地记录与平台状态可能不一致
- 需要清楚区分“平台完成”和“本地标记完成”

**前置条件**：

- 需要确认是否接受“本地标记不等于平台完成”

## P10：自动翻页 / 自动化阅读辅助

**类别**：高风险/需确认功能，高自动化，需谨慎确认

**价值**：

- 如果平台仅依赖简单翻页与停留，可能大幅减少机械操作

**风险**：

- 明显涉及平台规则风险
- 可能触发账号异常风险
- 稳定性极差，页面改版后容易失效
- 平台如果通过行为特征或接口校验判断，脚本很可能不稳定

**结论**：

- 不能作为默认目标
- 必须先确认边界要求和风险接受度
- 还必须拿到实际页面与请求链路后才能评估

## P11：自动触发打印 / 自动批量导出 PDF

**类别**：高风险/需确认功能，高自动化，需谨慎确认

**价值**：

- 理论上可进一步减少重复操作

**风险**：

- 浏览器对打印弹窗和下载行为限制很多
- 不同浏览器兼容性差
- 用户系统层“另存为 PDF”通常无法被油猴完全接管
- 稳定性差，维护成本高

**结论**：

- 不建议作为一期目标
- 更适合做“打印前辅助”而不是“自动完成导出”
