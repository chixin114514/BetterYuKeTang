# 建议的下一步工作顺序

建议按以下顺序推进，以降低返工和误判风险。

## 1. 先确认产品边界

明确项目只做：

- 页面增强
- 流程辅助

还是也允许：

- 轻度自动化
- 高自动化

这一步必须先定，因为它直接决定方案设计方向和风险控制方式。

## 2. 补齐页面样本

先提供最关键的页面信息：

- 课程内容列表页截图
- PPT 阅读页截图
- 打印入口截图
- 作业页截图
- 实验页截图

如果能补 HTML 片段更好。

## 3. 梳理真实用户流程

按实际使用过程，描述一次：

- 从登录到进入课程
- 从课程到打开 PPT
- 从 PPT 到完成阅读
- 从课件到打印导出 PDF
- 从课程到进入作业/实验

## 4. 确认一期范围

建议一期优先收敛为：

- 页面识别
- 内容分类
- 任务聚合与筛选
- PPT 阅读状态识别与可视化
- 打印/导出辅助
- 配置与日志基础设施

## 5. 再评估增强功能

等页面结构明确后，再决定是否加入：

- 快捷面板
- 未完成内容快速导航
- 本地标记
- 课程级概览

## 6. 最后再单独评估高自动化功能

只有在明确接受风险、且拿到足够页面与行为数据之后，才讨论：

- 自动翻页
- 自动触发阅读进度
- 批量自动化处理

## 建议优先回复的内容

现阶段最建议先补两部分内容：

- 对“自动化边界”的明确态度
- 至少一组课程列表页 + PPT 阅读页 + 打印入口页截图/说明
