# 长江雨课堂油猴插件项目分析文档

## 简介

本目录用于归档和持续维护“长江雨课堂使用体验优化”Tampermonkey（油猴）插件项目的前期分析材料。

当前文档内容聚焦于以下事项：

- 项目目标总结
- 已知需求梳理
- 功能模块拆分
- 功能扩展建议
- 各功能价值与风险说明
- 当前信息缺口
- 需要向需求方确认的问题清单
- 建议的下一步工作顺序

现阶段文档仅用于需求分析、产品规划、技术可行性评估与后续沟通准备，不包含代码实现、伪代码或脚本设计细节。

## 文件夹用途

该文件夹用于：

- 保存项目的结构化分析结论
- 便于后续补充页面截图、HTML 片段、交互流程和风险边界说明
- 支持后续继续细化为 MVP 范围、功能优先级表、技术方案文档和实施计划

## 文件清单

- `README.md`
  - 文档目录说明、用途说明、文件列表和推荐阅读顺序
- `01-project-goals-and-known-requirements.md`
  - 项目目标总结与已知需求梳理
- `02-functional-modules.md`
  - 项目可拆分的功能模块、职责边界与模块定位
- `03-feature-suggestions-by-priority.md`
  - 建议增加的功能列表，按优先级排序，并标注类别、场景、难点和前置条件
- `04-feature-value-and-risk.md`
  - 各类功能的价值、适用性和风险总结，包含高风险能力边界提醒
- `05-information-gaps.md`
  - 当前分析所需但尚未掌握的信息缺口
- `06-questions-to-confirm.md`
  - 继续推进分析前必须先确认的关键问题清单
- `07-next-steps.md`
  - 建议的下一步推进顺序和协作方式

## 推荐阅读顺序

建议按以下顺序阅读：

1. `01-project-goals-and-known-requirements.md`
2. `02-functional-modules.md`
3. `03-feature-suggestions-by-priority.md`
4. `04-feature-value-and-risk.md`
5. `05-information-gaps.md`
6. `06-questions-to-confirm.md`
7. `07-next-steps.md`

这样可以先建立项目边界和目标，再进入功能设计、风险判断和后续信息采集准备。
