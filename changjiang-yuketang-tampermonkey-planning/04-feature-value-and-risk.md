# 每个功能的价值与风险说明

## 核心功能

建议纳入一期：

- 课程/页面识别
- 内容分类
- 课程内任务聚合与筛选
- PPT 阅读状态识别与可视化
- 打印入口识别与导出辅助
- 配置管理
- 日志与兼容性处理

**价值**：

- 落地性强
- 风险可控
- 对学习效率提升最明显
- 油猴脚本非常适合做这类页面增强与流程辅助

**风险**：

- 依赖页面结构稳定性
- 依赖真实页面样本
- 如果平台是 SPA，监听机制设计不好会导致功能失效

## 增强功能

建议作为 1.5 期或二期：

- 阅读页快捷操作面板
- 未完成内容快速导航
- 课程级学习概览
- 本地标记/笔记类能力
- 导出前提示面板

**价值**：

- 用户体验提升明显
- 技术风险中等
- 通常不直接碰平台核心业务逻辑

**风险**：

- 容易因为 UI 层变动而需要适配
- 如果交互太重，会增加维护成本

## 可选功能

建议视实际页面情况决定：

- 批量打开待处理内容
- 本地学习记录增强
- 自定义快捷键/自定义面板布局

**价值**：

- 对重度使用者有帮助
- 但不是第一优先级

**风险**：

- 投入和收益可能不对等
- 部分功能依赖用户习惯，不一定刚需

## 高风险 / 需确认功能

现阶段不要默认纳入：

- 自动翻页
- 自动刷阅读进度
- 自动提交型操作
- 自动批量导出 PDF
- 自动化点击平台关键流程

**主要风险**：

- `平台规则风险`
  - 可能触碰平台使用规则
- `账号风险`
  - 存在被识别为异常行为的可能
- `稳定性风险`
  - 依赖接口、事件、埋点逻辑，极易失效
- `页面变更风险`
  - DOM 或前端框架稍改就坏
- `维护成本风险`
  - 每次平台更新都要重新适配
